glob-quote by hholzgra

Native quoting helper function for glob() patterns

There may be situations where path names that contain glob() meta characters like '[' should be taken by glob() "as is" instead of interpreting the meta characters. The new glob_quote() function will safely escape all glob() meta charaters so that e.g.: glob(glob_quote(getcwd()).DIRECTORY_SEPARATOR."*.txt"); will return the expected results even if the current directory path contains names that have meta characters like [, ], ? etc. in them.

Dependencies

None

Install

Build PHP as usual. glob() and glob_quote() are part of the "standard" extension, making them available on all unixoid systems with glob() in their libc and on MS Windows.
